# 小学单课 HTML 课件生成器

## Overview
把一节课的教案、教材页和课堂目标翻成可直接展示的 HTML 课件。核心原则：先判定范围，再补齐关键缺口，再按学科和课型做出像课堂而不是像普通网页的成品。

## When to Use
- 用户要做小学 1–3 年级语文、数学、英语课件
- 用户明确要“单课”“一课一做”“HTML 展示页”
- 用户给了教案、教材页、参考网页、样例 HTML，想生成接近完成度的课件
- 用户只给部分材料，希望先少量追问关键问题
- 用户同时要单文件 HTML 或 HTML + assets

**Do not use when**
- 用户只是要分析教案，不生成课件
- 用户要的是 PPT、PDF、表格、题库、OCR、翻译或整单元方案
- 用户在做非小学语数英的通用网页开发

## Quick Reference
| 情况 | 先做什么 | 输出重点 |
|---|---|---|
| 只有教案 | 先推断课型和课堂结构，再少量追问 | 大纲先行，补齐缺口 |
| 有教案 + 教材页 | 抓课题、目标、重点、活动顺序 | 页面内容与教材一致 |
| 有样例 HTML | 学完成度和展示感，不照抄布局 | 结构借鉴，视觉重做 |
| 要单文件 | 合并资源、减少外链 | 可直接打开展示 |
| 要 HTML + assets | 主文件清楚引用外部资源 | 易维护、易替换 |

## Decision Flow
```text
材料是否足够？
├─ 足够 → 直接定课型、出大纲、做 HTML
└─ 不够 → 只问 2–4 个最关键问题
        └─ 仍缺信息 → 基于教案合理推断，不要反复追问
```

## Implementation
### 1) 先判定范围
- 只做一课，不扩成单元，不混多个课时。
- 如果用户一次给多课，先锁定当前这一课。

### 2) 先理解材料
优先提取这些信息：
- 年级、学科、课题
- 教学目标、重点、难点
- 课堂活动顺序
- 练习/互动形式
- 是否有教材页或参考 HTML 可校准

**材料使用顺序：** 教案 > 教材页 > 参考 HTML/网页 > 其他素材。

### 3) 只问最关键的问题
只问会明显影响页面结构的问题。常见只需要问：
- 课题是否有明确版本/页码
- 这节课更偏新授、练习、复习还是展示
- 是否要单文件，还是 HTML + assets
- 是否需要互动按钮、翻翻卡、对话框、练习题等

**Rules:**
- 问题一般控制在 2–4 个。
- 问完就停，能推断的不要继续追问。
- 追问要围绕材料本身，不要泛泛问“还有什么要求”。

### 4) 先给课件大纲
输出顺序固定：
1. 课件大纲
2. HTML 成品
3. 简短生成说明

大纲建议包含：
- 导入
- 新授/讲解
- 练习/互动
- 总结/拓展

### 5) 再生成 HTML
- 默认做成课堂展示页，不要像普通博客、文档页或表单页。
- 页面要有清晰分区、节奏感和板块层次。
- 内容密度要接近高完成度示例，而不是空壳模板。
- 如果有练习，尽量把练习做成课堂中可读、可点、可展示的形式。

### 6) 最后补生成说明
只用很短的说明讲清楚：
- 采用了什么风格
- 为什么适合这个学科/课题
- 后续最值得改哪几处

## Subject and Lesson-Type Strategies
### 语文
- 适合故事感、情境感、文本层次、插图/卡片式排版
- 识字课：字形、读音、组词、书写分区清晰
- 阅读课：情境导入、段落理解、人物/事件/情感线索明显
- 朗读/复习课：节奏更轻，重点更突出

### 数学
- 适合步骤清晰、卡片分区、图示、操作演示、推导过程可见
- 口算/计算：步骤拆得细，错因和方法对比明显
- 概念课：定义、例子、反例、练习分块
- 应用题：题意、分析、列式、检验分层展示

### 英语
- 适合词汇卡、情境对话、句型替换、音形义联动
- 词汇课：图词对应、发音、跟读、分类更重要
- 对话课：角色、场景、问答轮次清楚
- 复习课：轻快但不杂乱，卡片和互动更灵活

### Same subject, different topic
同一学科也要变体。识字、阅读、口算、应用题、对话、复习课的视觉组织都可以不同，不要一套模板套到底。

## Visual System
- 语文：更温暖、有情境、留白适中
- 数学：更清楚、更结构化、强调对比和步骤
- 英语：更轻快、更多卡片感、颜色明亮但不花
- 统一要求：
  - 标题层级清晰
  - 卡片/模块边界明确
  - 课堂节奏能一眼看出来
  - 颜色服务内容，不抢内容

**Anti-patterns:**
- 一整页同样的方块堆叠
- 过度装饰但内容空
- 视觉统一到看不出学科差异
- 把教材内容直接搬成普通网页

## Output Protocol
Always follow this order:
1. `课件大纲`
2. `HTML 成品`
3. `生成说明`

### HTML rules
- 优先保证可直接展示
- 单文件模式：尽量把样式、脚本、内容收进一个文件
- HTML + assets：主文件清楚引用外部资源，资源命名短而直观
- 如果用户没选模式，按当前材料复杂度选择最稳的一种，并说明原因

## Quality Checks
Before finalizing, check:
- 是否只覆盖一课
- 是否和教案目标对齐
- 是否像课堂展示页，而不是通用网页
- 是否有学科/课型差异
- 是否内容过空、过挤、过于模板化
- 是否误把参考 HTML 当成要照抄的模板
- 是否少量追问后就停止，而不是反复问

## Common Mistakes
| Mistake | Fix |
|---|---|
| 把一课做成整单元 | 只锁定当前课题 |
| 追问太多 | 只问最影响结构的 2–4 个问题 |
| 语文/数学/英语都长得一样 | 按学科和课型重做版式 |
| 参考 HTML 被照抄 | 只学完成度和组织感 |
| 内容够了还继续问 | 直接进入大纲和 HTML |
| 页面像文档不是课件 | 加强分区、节奏和课堂感 |

## Reminder
先把一课讲清楚，再把它做漂亮。课件不是模板拼接，而是把课堂逻辑翻成可展示的 HTML。
