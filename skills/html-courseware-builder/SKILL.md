# 小学 HTML 课件生成器

## 它做什么
把教案、教材页和课堂要求转成一份可直接展示的单课 HTML 课件。优先保证课堂可用性、视觉完成度和学科适配，不把多个课时混在一起。

## 适用场景
- 用户要做 1–3 年级语文、数学、英语课件
- 用户要单课 HTML、互动式 HTML，或 HTML + assets 输出
- 用户只给了教案，希望先补问关键信息再生成
- 用户拿出教材页、教案、样例 HTML，希望生成风格接近但不照抄
- 用户需要课件大纲、页面结构、生成说明一起交付

## 工作流
1. **先判断范围**
   - 只做一课。
   - 如果用户一次想做多课，先拆成当前这一课，再继续。

2. **先读材料**
   - 有教案、教材页、样例 HTML 就先看。
   - 如果有教学材料仓库，优先从材料里找对应年级、学科、主题的内容再下结论。
   - 重点抓：年级、学科、课题、教学目标、课堂活动、板书/练习要求、已有风格参考。
   - 必要时可参考 `assets/examples/` 下的两个 HTML 样例作为质量基准，理解完成度、信息密度与交互感，但不要照抄结构。

3. **只问最关键的问题**
   - 信息不够时，先问最能决定课件结构和风格的问题。
   - 通常优先问：年级、学科、课题、课堂目标、可用材料、输出模式、是否需要互动。
   - 问题要少而准；一般控制在 2–4 个，够生成就停，不要把老师问烦。

4. **先给大纲，再给成品**
   - 先输出《课件大纲》，再生成 HTML。
   - 大纲要体现课堂节奏：导入、讲解、练习、互动、总结。
   - 如果用户只要大纲，也可以停在大纲，但不要省略这一步。

5. **再生成 HTML**
   - 默认输出一份完成度高、可以直接展示的课件。
   - 如果用户选择“单文件模式”，把所有内容尽量收进一个 HTML。
   - 如果用户选择“HTML + assets”，把主 HTML 和外部资源分开列清楚，命名要简单可维护。

6. **最后补生成说明**
   - 用很短的说明讲清楚：
     - 为什么用了这种风格
     - 哪些设计是根据学科/主题定制的
     - 如果老师后续想改，最该改哪几处

## 输出规范
- 永远围绕“单课”交付，不扩成整单元
- 永远保留课堂展示感，避免像普通博客页
- 永远让风格跟学科、主题走，不用一套模板套所有内容
- 永远优先保证老师能直接拿去用
- 永远把课件大纲放在 HTML 之前
- 永远把生成说明放在最后

## 风格策略
- **语文**：更适合故事感、情境感、插图感、文本层次分明的版式
- **数学**：更适合清晰步骤、卡片分区、图形化演示、推导过程可见的版式
- **英语**：更适合词汇卡、情境对话、音形义联动、轻快但不杂乱的版式
- **同一学科不同主题**：要继续变体。比如识字、阅读、口算、应用题、对话课、复习课，视觉组织都可以不同
- 参考 HTML 的价值在于完成度和审美，而不是复制布局

## 材料使用原则
- 教案是最低输入
- 如果材料不足，就根据教案先推断课堂结构，再补问关键细节
- 教材页、样例 HTML、练习样本都可以用来校准内容和风格
- 不要为了“更完整”硬塞跨课内容、跨单元内容或无关活动

## 记住
先把一课讲清楚，再把它做漂亮。
课件不是模板拼接，而是把课堂逻辑翻成可展示的 HTML。
